# Hackathon Judging & Scoring Matrix


| Category                                 | Requirement / Capability                                                                                            | Scoring Weight | Notes / Examples                                                                                                                  |
| ---------------------------------------- | ------------------------------------------------------------------------------------------------------------------- | -------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| **Quality and Completeness of Solution** | **35% total, broken down into:**                                                                                    | **35%**        | Judges evaluate end-to-end completeness, coherence, and correctness of deployed system.                                           |
| • Core System Functionality              | All trading mechanics implemented (buying from supply, bidding, accepting/rejecting trades, cash/inventory updates) | 10%            | Trades execute correctly; cash locked/released as required; inventory updates properly.                                           |
| • Valuation Mechanics                    | Intrinsic value formula applied correctly; age, health, desirability updated; expired pets handled                  | 8%             | Intrinsic value formula verified; ±5% variance applied; end-of-life pets remain in inventory.                                     |
| • Trader UI & Market View                | Trader panels, market view, new supply, listing details displayed correctly                                         | 5%             | Panels per Trader; market shows listings, asking price, most recent trade, new supply count; optional sorting/filtering is bonus. |
| • Analysis / Drill-Down View             | Correct display of pet fundamentals for analysis                                                                    | 6%             | Age, health, desirability, maintenance, intrinsic value visible; used by Traders for decisions.                                   |
| • Leaderboard                            | Portfolio values displayed correctly; updates in real-time                                                          | 6%             | Aggregate cash, locked cash, intrinsic value of pet inventory; used by Traders to strategize.                                     |
| **Use of AI Tools**                      | Depth, breadth, intentionality of AI usage across lifecycle (design, implementation, testing, deployment, etc.)     | **25%**        | Evidence via prompts, iterations, artifacts; judges evaluate reasoning, refinement, and transparency in AI-assisted decisions.    |
| **Components Completed**                 | Coverage of key system components required for a working SaaS solution                                              | **15%**        | Judges will assess both **presence and meaningful integration** of components. Subcomponents include:                             |
| • User Interface (Front End)             | Functional UI supporting core user flows; alignment with UX designs; basic state handling                           | 3%             | Correct rendering of Trader panels, market view, analysis, notifications, leaderboard.                                            |
| • Back-End Services / APIs               | Working APIs implementing core business logic; clear separation of concerns                                         | 3%             | Correct handling of trades, bids, inventory, cash, valuations, notifications.                                                     |
| • Data Model & Persistence               | Reasonable data structures; persistence layer aligned with system behavior                                          | 2%             | Proper tracking of pets, traders, bids, and market state; optional: in-memory or database.                                        |
| • System Test Coverage (Defined)         | AI-assisted identification of system-level test cases and edge cases (execution optional)                           | 2%             | Includes happy and non-happy paths; test cases linked to requirements.                                                            |
| • Infrastructure Definition              | Clear definition of runtime environment; IaC preferred                                                              | 1%             | Environment assumptions, service boundaries, cloud platform (AWS, Azure, GCP) configuration.                                      |
| • Environment Configuration              | Configuration of services, networking, permissions, and environment variables                                       | 1%             | Sensible defaults; consistency with architecture; security awareness.                                                             |
| • Deployment Pipeline                    | Defined build and deployment flow (manual or automated CI/CD)                                                       | 2%             | Pipeline clarity, repeatability, alignment with infrastructure and architecture.                                                  |
| • Operational Awareness (Optional Bonus) | Logs, metrics, monitoring definitions demonstrating awareness of runtime behavior                                   | 1%             | Optional observability; higher score if demonstrated effectively.                                                                 |
| **Quality of Presentation**              | Clarity, structure, ability to explain decisions and tradeoffs                                                      | **15%**        | Judges evaluate reasoning, explanation of tradeoffs, AI tool usage walkthrough, and live demo effectiveness.                      |
| **Team Voting (Peer Signal)**            | Peer evaluation by other teams                                                                                      | **10%**        | Peer voting used as input signal, but outcome primarily determined by judge evaluation.                                           |
