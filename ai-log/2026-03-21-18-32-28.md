## Response Summary

Explained the ECR state conflict: ECR repositories are named without environment suffix (`pts-trading-api`, `pts-lifecycle-lambda`), so both dev and demo Terraform states try to create the same AWS resources. The demo apply will not overwrite — it will fail with `RepositoryAlreadyExistsException`. Read `terraform/main.tf`, `terraform/modules/ecr/main.tf`, `.github/workflows/initial-setup.yml`, `.github/workflows/deploy-demo.yml`, and `terraform/environments/demo.tfvars` to confirm the design intent (shared ECR, env-differentiated by image tag) versus the broken Terraform implementation. Proposed fix: add `count = var.environment == "dev" ? 1 : 0` on ECR resources and `data` source fallbacks for demo, and offered to dispatch `senior-devops-engineer` to implement it.
