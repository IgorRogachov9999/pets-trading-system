## Response Summary

Explained the frontend build storage mechanism by referencing `initial-setup.yml` and `post-merge-build.yml`. Confirmed that every build is stored under both `frontend/{git-sha}/` (permanent, per-commit) and `frontend/latest/` (overwritten each time) in the builds S3 bucket. Explained that `deploy-dev.yml` and `deploy-demo.yml` accept an `image_tag` input (defaulting to `latest`) which can be set to any SHA for targeted deployments or rollbacks. Noted that the same pattern applies to backend Docker images in ECR, and that there is no retention/cleanup policy (acceptable for hackathon).
