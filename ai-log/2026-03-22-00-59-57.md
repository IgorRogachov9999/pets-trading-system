## Problem

`deploy-dev.yml` triggers via `workflow_run` from 3 build workflows. If two builds complete in quick succession, two deploy-dev runs start simultaneously and fight over the Terraform DynamoDB state lock, causing `ConditionalCheckFailedException`.

## Fix

Read `.github/workflows/deploy-dev.yml` first, then add a `concurrency:` block at the workflow top level (same indentation as `on:`, `permissions:`, `env:`, `jobs:`):

```yaml
concurrency:
  group: deploy-dev
  cancel-in-progress: false
```

`cancel-in-progress: false` ensures queued runs complete in order rather than being dropped.

- `cancel-in-progress: false`
