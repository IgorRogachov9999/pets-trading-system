## Response Summary

Diagnosed two bugs via AWS CLI and CloudWatch logs. (1) ECS container health check used `curl -f http://localhost:8080/api/health` but `curl` is not installed in `dotnet/aspnet:10.0` — the CMD-SHELL check always failed, causing ECS to kill the container after startPeriod+retries; fixed by registering task definition revision 2 with `wget` via AWS CLI (immediate) and updating `terraform/modules/ecs/main.tf` line 131 (code). (2) `VITE_API_BASE_URL` was never set at build time in `initial-setup.yml`, so all frontend API calls fell back to same-origin CloudFront which returns HTML; fixed by moving AWS credentials before `npm run build` and adding a step that queries `aws apigateway get-rest-apis` to construct and export `VITE_API_BASE_URL`. Both changes committed as `1ec2014` and pushed to `fix/Fix-pipeline-2`.
