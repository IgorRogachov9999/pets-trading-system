# Log Entry - 2026-03-22 00:04:03 UTC

## User Request

what does it mean :8 on the task definition and why it was not updated after pushing new image?

## Response Summary

Explained that :8 is the ECS task definition revision number — an immutable snapshot of container configuration registered in AWS. Explained two reasons the revision doesn't increment on image push: (1) lifecycle { ignore_changes = [task_definition] } in the Terraform ECS module prevents Terraform from updating the service's task definition, and (2) the deploy pipeline uses --force-new-deployment without --task-definition, which restarts containers with the new ECR image but keeps the same revision number. Confirmed :8 is correct and expected — the new curl-enabled image runs inside it even though the revision number hasn't changed.
