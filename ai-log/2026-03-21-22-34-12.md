# Log Entry - 2026-03-21 22:34:12 UTC

## User Request

the ecs tasks are failing to start and also healthcheck is unhelaty, find and fix the issue, here are some logs from the tasks [logs showing /api/v1/health returning 200]

## Response Summary

Identified two-layer root cause: (1) NLB health check path was already fixed to /api/v1/health by a previous pipeline run (explaining logs showing 200), but (2) the ECS service ignores task definition changes via `lifecycle { ignore_changes = [task_definition] }`, so Terraform registering a new task definition with the corrected container health check path never gets applied to running tasks. The deploy steps were calling `--force-new-deployment` without specifying the new task definition ARN, so tasks kept relaunching with the OLD container health check (`wget /api/health` → 404 → container UNHEALTHY → killed). Fixed by updating the "Deploy ECS" step in initial-setup.yml, deploy-dev.yml, and deploy-demo.yml to resolve the latest registered task definition ARN with `aws ecs describe-task-definition` and pass it explicitly via `--task-definition` to `aws ecs update-service`.
